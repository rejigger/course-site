Weekly exercises

Week 1 — Go Fundamentals

Day 1: Introduction to Go

Go Tour: Welcome
Exercise: Complete the first 2 “A Tour of Go” sections.

Day 2: Variables, Types, and Control Structures

  • Go by Example: Basics
    Exercise: Write a small program using if, for, and variables.

Day 3: Functions and Multiple Return Values

  • Go Official: Functions
    Exercise: Write a function that returns the sum and product of two numbers.

Day 4: Structs and Methods

  • Go Official: Structs

  • Go Official: Interfaces
    Exercise: Create a struct for License with methods for formatting.

Day 5: Error Handling

  • Go Official: Error Handling
    Exercise: Modify your function to return an error if inputs are negative.

Day 6: Working with Packages and Modules

  • Go Modules Tutorial
    Exercise: Create a small Go module with one package and import it in your main program.

Day 7: Review / Mini Project

  • Combine structs, functions, and error handling into a small console app (like tracking “licenses” in memory).

Week 2 — Web & Database Fundamentals

Day 8: HTTP Servers in Go

  • Go by Example: HTTP Servers
    Exercise: Serve a “Hello HTMX + Go” page locally.

Day 9: HTML Templates

  • Go Templates

  • Go by Example: Templates
    Exercise: Render a simple HTML page using Go templates.

Day 10: PostgreSQL Basics

  • PostgreSQL Tutorial
    Exercise: Create a licenses table and insert some sample data.

Day 11: Connecting Go to PostgreSQL (pgx)

  • pgx Documentation

  • Go PostgreSQL Tutorial
    Exercise: Write a Go program that queries your licenses table and prints results.

Day 12: Querying with pgx

  • pgx QueryRow & Query
    Exercise: Fetch all licenses and scan into a struct slice.

Day 13: HTML Templates + Database Data

  • Go Templates with Data
    Exercise: Render license data from your database in an HTML table.

Day 14: Review / Mini Project

  • Build a page that shows all license checkouts in a table using templates and PostgreSQL.

Week 3 — HTMX, Charts, and Deployment

Day 15: Introduction to HTMX

  • htmx.org Guide
    Exercise: Make a simple HTMX request that swaps a <div> content.

Day 16: HTMX Attributes & Events

  • htmx.org Attributes

  • htmx.org Events
    Exercise: Add a dropdown that updates a table via HTMX.

Day 17: Chart.js Basics

  • Chart.js Getting Started
    Exercise: Render a simple line chart with hardcoded data.

Day 18: Dynamic Chart Updates

  • Chart.js Update Data
    Exercise: Update your chart when dropdown changes.

Day 19: Full Stack Integration — Go + HTMX + Chart.js

  • htmx.org Server Examples
    Exercise: Wire your HTMX dropdown to update both the highlights table and chart.

Day 20: Deployment to Railway

  • Deploying Go on Railway

  • Railway Environment Variables
    Exercise: Deploy your project to Railway.

Day 21: Review & Polish

  • Test dropdown filtering, chart updates, highlights stats.

  • Make final adjustments and ensure everything works both locally and on Railway.
